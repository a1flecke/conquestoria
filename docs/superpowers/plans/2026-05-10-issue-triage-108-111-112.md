**Player Truth Table**

| Before | Action | Immediate visible result |
|---|---|---|
| Start screen shows saved games | Open saved games | Saved campaigns/epics appear as first-level rows, named from `gameTitle` |

**Queue And ETA Checklist**

- Active item: latest save for each epic is shown on the epic row.
- Queued follow-ups: not applicable.
- ETA/order text: rows sort newest turn first; deletion preserves ordering.
- Retention: each epic detail shows at most five newest saves total, counting autosaves and manual saves with that `gameId`; older saves remain in storage unless the user explicitly deletes/overwrites them.
- Invalid queued item behavior: stale autosave metadata with missing payload remains pruned by existing loadable autosave logic.

**Misleading UI Risks**

- An `epic` must mean one `gameId`; same title with different `gameId`s must remain separate.
- Manual saves with a `gameId` should appear inside the matching epic instead of only in a flat list.
- The UI must never show more than five save rows inside one epic detail, even if the backing store has more manual saves for that `gameId`.
- Unlimited epics remain allowed; the five-row limit applies within each epic, not across all campaigns.
- The save panel must consume `listSaveEpics()` in start mode instead of duplicating grouping logic locally.

- [x] **Step 3: Implement grouped save metadata**

In `src/storage/save-manager.ts`, export a grouped type/helper. `listSaveEpics()` is the only grouped save source; UI code must not rebuild a different grouping contract.

```ts
export interface SaveEpic {
  gameId: string;
  title: string;
  latestTurn: number;
  latestPlayed: string;
  gameMode: SaveSlotMeta['gameMode'];
  playerNames?: string[];
  saves: SaveSlotMeta[];
}

export function groupSaveEpics(saves: SaveSlotMeta[]): SaveEpic[] {
  const groups = new Map<string, SaveSlotMeta[]>();
  for (const save of saves) {
    const gameId = save.gameId ?? save.id;
    groups.set(gameId, [...(groups.get(gameId) ?? []), save]);
  }

  return [...groups.entries()]
    .map(([gameId, entries]) => {
      const sorted = [...entries].sort(compareSaveMeta);
      const newestFive = sorted.slice(0, 5);
      const latest = newestFive[0]!;
      return {
        gameId,
        title: latest.gameTitle ?? latest.name,
        latestTurn: latest.turn,
        latestPlayed: latest.lastPlayed,
        gameMode: latest.gameMode,
        playerNames: latest.playerNames,
        saves: newestFive,
      };
    })
    .sort((left, right) => right.latestTurn - left.latestTurn || Date.parse(right.latestPlayed) - Date.parse(left.latestPlayed));
}

export async function listSaveEpics(): Promise<SaveEpic[]> {
  const saves = await listSaves({ includeAutoSave: true });
  return groupSaveEpics(saves);
}
```

Change signatures in `src/systems/improvement-system.ts`:

```ts
interface WorkerActionEligibilityOptions {
  isCityTile?: boolean;
}

export function canBuildImprovement(
  tile: HexTile,
  type: BuildableImprovementType,
  completedTechs: string[] = [],
  ownerId?: string,
  options: WorkerActionEligibilityOptions = {},
): boolean {
  if (options.isCityTile) return false;
  const definition = IMPROVEMENT_DEFINITIONS[type];
  if (!definition) return false;
  if (ownerId && tile.owner !== ownerId) return false;
  if (tile.improvement !== 'none') return false;
  if (!definition.validTerrains.includes(tile.terrain)) return false;
  if (definition.requiresRiver && !tile.hasRiver) return false;
  if (definition.requiredTech && !completedTechs.includes(definition.requiredTech)) return false;
  return true;
}
```

Make the same option apply to `canDrainSwamp` and `getAvailableWorkerActions`:

```ts
export function canDrainSwamp(
  tile: HexTile,
  ownerId?: string,
  options: WorkerActionEligibilityOptions = {},
): boolean {
  if (options.isCityTile) return false;
  if (ownerId && tile.owner !== ownerId) return false;
  return tile.terrain === 'swamp' && tile.improvement === 'none';
}

export function getAvailableWorkerActions(
  tile: HexTile | undefined,
  completedTechs: string[] = [],
  ownerId?: string,
  options: WorkerActionEligibilityOptions = {},
): WorkerActionType[] {
  if (!tile) return [];
  const actions: WorkerActionType[] = [];
  for (const type of Object.keys(IMPROVEMENT_DEFINITIONS) as BuildableImprovementType[]) {
    if (canBuildImprovement(tile, type, completedTechs, ownerId, options)) actions.push(type);
  }
  if (canDrainSwamp(tile, ownerId, options)) actions.push('drain_swamp');
  return actions;
}
```
